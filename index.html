# AI Minecraft Mod Builder — Full Website + Mod Template This single-file repository contains a complete starter website (frontend + backend) and a ready-to-use Minecraft Forge mod template plus a GitHub Actions workflow that will build the mod JAR automatically on push. Use it to run an "AI-driven" mod generator web app locally or on a server and get an automated build producing the .jar. --- ## Project layout (all files included below) - README.md - frontend/ (React app) - package.json - src/App.jsx - src/index.jsx - public/index.html - backend/ (Node + Express) - package.json - server.js - templates/forge-1.20.1/ (mod template) - build.gradle - gradle/ (wrapper - instructions included to generate wrappers locally) - settings.gradle - src/main/java/com/example/mod/ExampleMod.java - src/main/resources/META-INF/mods.toml - src/main/resources/assets/examplemod/lang/en_us.json - .github/workflows/build-mod.yml (GitHub Actions to build JAR) --- ## README.md ```markdown # AI Minecraft Mod Builder This project is a starter website for generating Minecraft Forge mods from simple inputs. It provides: - A React frontend where users enter mod metadata and feature requests. - A Node/Express backend that generates a mod template (Forge 1.20.1 sample) using the inputs. - A GitHub Actions workflow that builds the mod JAR automatically on push using Gradle. **Important:** This repository contains source code and a Gradle project template. I cannot compile the mod into a .jar inside this web preview. However, follow the instructions below or use the included GitHub Actions workflow to build the JAR automatically. ## Quick start (local) 1. Clone the repo. 2. Install frontend deps: ```bash cd frontend npm install npm run dev ``` 3. Start backend in another terminal: ```bash cd backend npm install node server.js ``` 4. Open http://localhost:5173 (or the frontend dev port). Use the UI to generate mod sources. 5. To build the mod JAR locally, go to `backend/templates/forge-1.20.1/` and follow Forge MDK instructions: - Download Forge MDK for the matching Minecraft/Forge version, or initialize Gradle wrapper locally using `gradle wrapper`. - Run `./gradlew build` (on Windows `gradlew.bat build`). The JAR will appear in `build/libs/`. ## Quick start (automated build via GitHub Actions) 1. Push the repo to GitHub. 2. The workflow `.github/workflows/build-mod.yml` will run on push and produce a build artifact (the mod jar). 3. Download the artifact from the Actions run. ## Notes & security - The generator writes files server-side in a temp workspace and returns a ZIP of generated source. The Actions workflow demonstrates how to run a build on push — adjust versions carefully. - Do not run arbitrary code from untrusted sources in production. filename=${modId}-source.zip`) const archive = archiver('zip') archive.on('error', err => { throw err }) archive.directory(tmp, false) archive.pipe(res) await archive.finalize() // Cleanup (fire & forget) setTimeout(()=> fs.remove(tmp).catch(()=>{}), 10000) }catch(err){ console.error(err) res.status(500).json({error: String(err)}) } }) app.listen(4000, ()=> console.log('Backend listening on http://localhost:4000')) ``` --- ## backend/templates/forge-1.20.1/build.gradle ```gradle plugins { id 'java' id 'net.minecraftforge.gradle' version '5.1.45' // example version — set appropriate plugin for target Forge } group = 'com.example' version = '__MOD_VERSION__' archivesBaseName = '__MOD_ID__' minecraft { version = "__MC_VERSION__-forge" } repositories { mavenCentral() } dependencies { // Add Forge dependencies if needed } jar { manifest { attributes 'Specification-Title': '__MOD_NAME__', 'Specification-Vendor': 'AI Mod Builder' } } ``` --- ## backend/templates/forge-1.20.1/settings.gradle ```groovy rootProject.name = '__MOD_ID__' ``` --- ## backend/templates/forge-1.20.1/src/main/java/com/example/mod/ExampleMod.java ```java package com.example.mod;
